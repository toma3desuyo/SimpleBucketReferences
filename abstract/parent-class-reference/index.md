---

## 概要
親クラス参照は派生クラス内から基底クラスのメソッドやプロパティへアクセスする機構であり、`super` などのキーワードで提供される。継承階層での再利用や基底の初期化に必須である。

## 使いどころ
- **コンストラクターの連鎖**: 基底の初期化ロジックを再利用する。
- **メソッドの拡張**: 親の処理を実行したうえで追加ロジックを差し込む。
- **プロパティの再利用**: 読み取り専用フィールドを親から取得する。

## 基本例
```pseudo
クラス 子 継承 親:
    コンストラクター(...):
        super(...)
        追加設定()

    メソッド 実行():
        結果 = super.実行()
        return 調整(結果)
```
- 親の処理を呼び出し追加の調整をする様子を示す。

## 応用パターン
### 複数継承の解決順序
```pseudo
クラス C(A, B):
    メソッド 実行():
        super().実行()  # メソッド解決順に従う
```
- 解決順序 (MRO) を理解し、期待通りに親メソッドが呼ばれるようにする。

### 親プロパティの公開
```pseudo
プロパティ 読み取り専用:
    return super.値
```
- 子で別名を与えたり検証を追加して露出する。

## 注意点
1. **再帰呼び出し**: super 呼び出しを忘れると基底処理がスキップされる。逆に super 内で同じメソッドを呼ぶとループになる言語もある。
2. **状態依存**: 親の初期化が完了する前に super を使うと未初期化状態に触れる。コンストラクターの順序を守る。
3. **多重継承の競合**: 同名メソッドが複数の親にある場合、明示的にどの親を呼ぶか指定する仕組みを用意する。
