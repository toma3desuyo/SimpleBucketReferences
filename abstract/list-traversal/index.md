## 概要
リストの走査は、全要素を順番に訪問して処理する行為である。反復の制御方法や途中での挙動を設計することで、性能と可読性を高められる。

## 走査方法
- **for ループ**: インデックスまたはイテレーターで走査。
- **foreach**: イテレーターが提供する順序のまま要素を受け取る。
- **ストリーム**: map/filter/reduce を組み合わせて宣言的に処理。

## 基本例
```pseudo
for index = 0 から リスト.長さ-1 まで:
    要素 = リスト[index]
    処理(要素)
```
- インデックスを利用して位置情報が必要な処理を行う。

## 応用パターン
### 走査と集計
```pseudo
合計 = 0
for 要素 in リスト:
    合計 += 要素.値
```
- 集約操作を走査と同時に実施する。

### 並列走査
```pseudo
チャンク = リスト.分割(スレッド数)
並列で for 各塊 in チャンク: 処理(塊)
```
- データを分割して複数ワーカーで処理し、結果を統合する。

## 注意点
1. **変更の扱い**: 走査中にリストを変更するとイテレーターが無効化される場合がある。コピーを作るか安全な API を使う。
2. **例外処理**: 途中で例外が出ると走査が途中で止まる。必要に応じて try/catch で包む。
3. **大規模データ**: 大きなリストを走査する際は早期終了条件を設けるか、ストリーミングで分割して処理する。
