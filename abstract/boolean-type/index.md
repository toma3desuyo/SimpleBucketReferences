## 概要
真偽値型は「真」と「偽」の 2 値のみを取り、条件分岐やループ制御、フラグ管理の基礎となる。多くの言語では `true`/`false` あるいは `真`/`偽` のリテラルを提供する。

## 特徴
- **二値論理**: 真偽値同士の論理演算（AND/OR/NOT）で条件を組み合わせる。
- **暗黙変換**: 0 や空文字を偽と解釈する言語もある。暗黙変換の規則を統一的に理解する。
- **格納コスト**: 実装上は 1 バイト以上使うが、ビットフィールドとして詰められるケースもある。

## 基本操作
```pseudo
有効 = 利用者入力 が 存在 かつ 利用者入力.確認済み == 真
もし 有効 なら:
    保存()
そうでなければ:
    警告を表示
```
- 真偽値は条件式の結果として得るほか、状態フラグとして構造体内に保持できる。

## 応用パターン
### 三値論理
```pseudo
状態 = { 真, 偽, 未設定 }
もし 状態 == 未設定 なら 既定を適用
```
- `null` などを加えて「未決定」を表現することで、ユーザー入力の有無を区別できる。

### ビットマスク
```pseudo
権限 = 0b000
権限 = 権限 OR 読み取り
もし (権限 AND 書き込み) != 0 なら ...
```
- 多数の真偽値を 1 つの整数で管理する最適化。読みやすさとトレードオフ。

## 注意点
1. **真偽値以外の値**: ループ条件に整数をそのまま用いると可読性が下がる。比較式を明示する。
2. **フラグ乱用**: 構造体にフラグが増えすぎると状態遷移が把握しにくくなる。列挙型や状態マシンへの置換を検討する。
3. **暗黙比較**: `if 値:` 形式を使う場合、`0` や `""` を意図的に真と扱いたいケースで混乱が生じる。言語仕様をチームで共有する。
