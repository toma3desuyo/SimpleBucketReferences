## 概要
クラスはデータ構造とその操作をひとまとめにした設計図であり、インスタンスを生成して再利用可能なオブジェクトを作る。カプセル化、継承、多態性といったオブジェクト指向の基盤となる。

## 構成要素
- **フィールド**: インスタンスが保持する状態。公開範囲と初期化ルールを定める。
- **メソッド**: 状態に対する操作。副作用の可否や例外を文書化する。
- **静的メンバー**: クラス全体に属するユーティリティや定数。`abstract/static-members/index.md` 参照。

## 基本例
```pseudo
クラス ユーザー:
    プロパティ id
    プロパティ 名前

    コンストラクター(id, 名前):
        self.id = id
        self.名前 = 名前

    メソッド 表示名():
        return self.id + ":" + self.名前
```
- コンストラクターで初期化し、公開メソッドで状態を操作する基本例。

## 応用パターン
### 継承
```pseudo
クラス 管理者 継承 ユーザー:
    メソッド 権限一覧(): return ["管理"]
```
- 共通ロジックを親クラスへまとめ、子クラスで拡張する。多重継承の有無を言語仕様に従う。

### インターフェース実装
```pseudo
インターフェース 送信可能:
    メソッド 送信(): 戻り値なし

クラス メール送信者 実装 送信可能:
    メソッド 送信(): ネットワーク送信
```
- 契約を明示し、依存側はインターフェース越しに扱う。

## 注意点
1. **過度な責務**: 単一責務原則を守り、1 クラスに多機能を詰め込みすぎない。
2. **循環参照**: 相互依存するクラスは結合度を上げる。抽象クラスやインターフェースを介して依存方向を整理する。
3. **テスト可能性**: 依存を外部から注入できるように設計し、状態を外部と同期させる手段（ゲッター/セッター）を用意する。
