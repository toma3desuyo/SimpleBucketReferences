---

## 概要
継承は基底クラスのメンバーを派生クラスへ受け継ぎ、差分だけを追加・変更できるメカニズムである。共通ロジックの再利用やポリモーフィズムを実現する。

## 種類
- **単一継承**: 1 つの親から継承する。構造が単純でダイヤモンド問題を避けやすい。
- **多重継承**: 複数の親を持てるが、メソッド解決順序を理解する必要がある。
- **インターフェース/トレイト**: 実装を持たない契約だけ継承し、必要なメソッドを実装側で提供する。

## 基本例
```pseudo
クラス 図形:
    メソッド 面積(): return 0

クラス 円 継承 図形:
    コンストラクター(半径)
    メソッド 面積(): return π * 半径^2
```
- 親クラスの契約を守りつつ、派生クラスで固有の実装を書く。

## 応用パターン
### メソッドのオーバーライド
```pseudo
クラス ロギングHTTP 継承 HTTPクライアント:
    メソッド 送信(要求):
        ログ(要求)
        return super.送信(要求)
```
- `super` を呼ぶことで親実装を再利用できる。

### 抽象クラス
```pseudo
抽象クラス ストレージ:
    抽象メソッド 保存(データ)

クラス ファイルストレージ 継承 ストレージ:
    メソッド 保存(データ): ...
```
- 抽象メソッドで派生クラスに実装を強制し、インターフェースを統一する。

## 注意点
1. **過度な継承**: 階層が深くなると理解が難しい。コンポジションで代替可能かを検討する。
2. **破壊的変更**: 親クラスの変更が子に波及する。互換性を保つためにテンプレートメソッドやフックを提供する。
3. **コンストラクターの順序**: 親の初期化が先に走る。派生クラスで未初期化のフィールドを参照しないよう注意。
