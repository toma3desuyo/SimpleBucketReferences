# フロントエンドフレームワーク概説

本稿では主要なフロントエンドフレームワークを概観し、採用の際に着目すべき特徴をまとめます。いずれもSPA/MPA/SSR/静的生成など多様なレンダリング戦略をサポートします。

## React
- コンポーネント指向と仮想DOMを採用するライブラリ。
- Hooks により状態管理・副作用制御を関数型で記述。
- Next.js などのメタフレームワークが豊富。

## Vue
- 漸進的フレームワーク。テンプレート構文と Composition API を提供。
- SFC (Single File Component) でテンプレート・ロジック・スタイルを同居。
- Nuxt によりSSR/静的生成の公式ソリューションが整備。

## Svelte
- ビルド時コンパイルで軽量なランタイムを実現。
- 宣言的リアクティビティ（`$:`）で少ない記述量。
- SvelteKit がルーティングやSSRを一体提供。

## Astro
- アイランドアーキテクチャで必要箇所のみをインタラクティブ化。
- 複数フレームワークのコンポーネントを混在可能（React/Vue/Svelte 等）。
- デフォルトは静的最適化重視でパフォーマンスに強い。

## Gatsby
- React ベースの静的サイトジェネレータ。
- GraphQL データレイヤとプラグイン群で多様なソースを統合。
- 画像最適化やプリフェッチを組み込み。

## Next.js
- React 公式推奨のフルスタックフレームワーク。
- App Router によりSSR/SSG/ISR/Streamingを柔軟に選択。
- API Routes や Route Handlers でバックエンドも同居。

## Nuxt
- Vue 向けフルスタックフレームワーク。
- ファイルベースルーティング、SSR/SSG/ISR を標準サポート。
- Nitro ベースでサーバーレス/エッジデプロイに対応。

## Remix
- Web標準重視のフルスタックフレームワーク。
- フォーム送信やルートごとの loader/action でデータ取得・更新を記述。
- 最小限のクライアントJSで高速レンダリングを志向。
