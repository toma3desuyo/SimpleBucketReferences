# サーバーサイドフレームワーク概説

主要なNode.js系サーバーサイドフレームワークの特徴をまとめ、選定時の観点を示します。軽量マイクロフレームワークからDIを備えた大型フレームワークまで多様です。

## Hono
- エッジ環境 (Workers/Pages/Functions) を想定した軽量フレームワーク。
- ミドルウェア互換が高く、`fetch` ベースのAPIで学習コストが低い。
- マルチランタイム（Node/Deno/Bun/Edge）を公式サポート。

## Express
- Node.jsで最も普及したミニマルフレームワーク。
- ミドルウェア資産が豊富で学習資料も多い。
- 既存コードベースや小規模APIで採用しやすい。

## Fastify
- 高性能と型安全性を重視。JSON Schema による検証を組み込み。
- プラグインシステムが強力で、エコシステムが拡大中。
- ロギングや計測が標準で扱いやすい。

## Koa
- async/await 前提のミドルウェアスタックを採用。
- シンプルな設計で自由度が高いが、周辺は自分で選定する前提。
- Express作者が開発。

## NestJS
- TypeScript前提のオピニオン付きフレームワーク。
- DIコンテナ、モジュール構造、デコレータベースのルーティングを提供。
- 大規模開発やマイクロサービスにも対応（GraphQL/WS/GRPCサポート）。

## AdonisJS
- Railsライクなフルスタック。認証、バリデーション、ORM(Lucid)をバンドル。
- 規約重視で一貫した開発体験を提供。
- SSR や静的生成にも対応する v6 の Blueprints を提供予定。

## Hapi
- 設定主導のプラグインフレームワーク。堅牢なバリデーション/認可を備える。
- 大規模APIや企業利用での実績があり、セキュリティオプションが充実。
- ルーティングや拡張ポイントが明確で保守しやすい。
